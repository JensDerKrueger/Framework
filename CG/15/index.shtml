Sheet 15: Hello OpenGL

Exercise: Hello OpenGL (20 points)

This and the following exercise sheets use OpenGL to render the scene. All OpenGL exercises must be implemented using OpenGL 3.2 Core Profile or higher. OpenGL 3 is the foundation of modern OpenGL; the fixed function pipeline and immediate mode commands are deprecated.

For more information on OpenGL, see the literature list from the lecture. The OpenGL Red Book and the Real-Time Rendering compendium are especially recommended. A quick overview of OpenGL and GLSL commands from version 3.2 onward is provided by the Quick Reference Cards.

Use the OpenGL Extensions Viewer to check whether your system supports OpenGL 3.2 or higher.

The sample program Hello OpenGL initially renders a simple white triangle. Extend the program so it matches the target image exactly: two colored triangles form a rectangle on a dark cyan background.

To solve this exercise:

  • Familiarize yourself with shaders, VBOs, and VAOs.
  • Adjust the data in the triangle array so one of the two triangles is rendered correctly.
  • Add additional geometry for the second triangle.
  • To color the two triangles differently, create a second shader program with a different fragment shader.

The colors are given as normalized RGB values: 

orange:     (0.9, 0.5, 0.2)
green:      (0.2, 0.9, 0.2)
background: (0.1, 0.15, 0.15)

Call glGetError after every OpenGL command to detect invalid commands or arguments immediately. According to the specification, OpenGL behavior is undefined in an invalid state, which can lead to hard-to-trace platform differences.
