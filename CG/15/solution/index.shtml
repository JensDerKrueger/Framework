Rectangle from Two Triangles

The starter code contains only one triangle. The solution replaces this geometry with two triangles that together form a rectangle in normalized device coordinates from -0.5 to 0.5. Both triangles share the diagonal from upper left to lower right.

The OpenGL objects are extended to arrays: two VBOs and two VAOs, one set per triangle. This is not the only possible solution, but it makes the mapping between geometry and draw call direct.

Shaders for the Two Colors

Both triangles use the same vertex shader because their positions are processed in the same way. For the different colors, the solution defines two fragment shader sources: one orange shader and one green shader.

Drawing

In draw, the framebuffer is cleared and then each triangle is drawn separately. Before each draw call, the solution binds the matching VAO and shader program. This gives the first triangle the orange color and the second triangle the green color.

Compile Shaders and Link Programs

setupShaders compiles the shared vertex shader and the two fragment shaders. It then links two programs: orangeProgram combines the vertex shader with the orange fragment shader, and greenProgram uses the green fragment shader. After linking, the individual shader objects can be deleted.

Geometry in VBOs and VAOs

In setupGeometry, two buffers and two vertex array objects are created. For each triangle, the matching VAO is bound, the corresponding VBO is filled with position data, and attribute 0 is configured as a three-component floating-point position.

This lets the rendering path later bind only the desired VAO and draw exactly one triangle with glDrawArrays(GL_TRIANGLES, 0, 3).
