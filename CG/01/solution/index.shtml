Exercise 1: Load OBJ Data

The starter code initially renders a manually specified sample triangle. The solution replaces that block in init with two calls to a shared helper function: one loads bunny.obj, the other loads the newly created cube.obj. Both resulting vectors are stored so the program can switch between the models later.

Convert Shared Vertices to Explicit Triangles

OBJFile stores the geometry in shared vertex arrays: positions and normals are stored once, while the triangles refer to them by index. For drawTriangles, the solution creates an explicit representation by iterating over all triangles and their indices, then writing the data to a new vector in the required order.

Each vertex stores position, color, and normal. As a simple color, the solution uses the vertex position plus 0.5 and an alpha value of 1.0. This follows the reference idea from the assignment, where the color depends on the position.

Exercise 2: Cube in OBJ Format

cube.obj describes a simple cube with eight shared vertices. The faces are given as twelve triangles, two triangles for each side of the cube. This satisfies the assignment requirement to use a shared-vertex representation.

Rendering and Switching

In draw, the camera, projection, and rotation from the starter code remain unchanged. The only difference is which data vector is drawn: if bCube is set, the cube is rendered; otherwise the bunny is rendered. The Spacebar toggles this state in keyboardChar.
