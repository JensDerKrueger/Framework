Sheet 1: Introduction

General

Upload your exercises to Moodle before the submission deadline. Late submissions receive 0 points, and submissions that do not compile also receive 0 points. Sample inputs and outputs are often provided; use them to test your program. Please do not add extra output unless the assignment explicitly asks for it. Submit one zip file named a1_Firstname_Lastname.zip. For this first sheet, a complete submission should contain main.cpp and cube.obj.

Compress all files you changed into one archive. If an exercise contains theoretical questions, include a text file, scan, or photo with your answers.

Setting Up the Development Environment

Follow these steps to set up your exercise environment:

Windows users:

Download the latest Visual Studio IDE (not Visual Studio Code).

Select "Desktop development with C++" during installation.

Mac users:

Install Xcode, and also install GLEW and GLFW.

Linux users:

Install the required tools with:

sudo apt install build-essential libglew-dev libglfw3-dev

Download the Framework and Exercise Code

Download the exercise framework and starter files from GitHub at https://github.com/JensDerKrueger/CG, or clone the repository with:

git clone https://github.com/JensDerKrueger/CG

Exercise 1: Hello OBJ (10 points)

Edit main.cpp in the Hello_OBJ folder so that it renders the OBJ model of a bunny. The class OBJFile reads bunny.obj. Your task is to convert the indexed representation into an explicit vertex representation.

Exercise 2: Create a Cube (5 points)

Create a cube.obj file that describes a cube. Store only position data and topology.

Implement the necessary code in init to load the cube data, and add a keyboard shortcut that switches between the bunny and the cube when the user presses the Spacebar.

A correct solution should look roughly like this:

Loading
Press the Spacebar to switch between the bunny and the cube.
