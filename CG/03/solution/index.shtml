Exercise 1: Evaluate Polynomial Curves

Hermite, Bezier, and B-spline segments are described in the starter code by different geometry matrices. The solution wraps their shared evaluation logic in computePoly. For a parameter t, it first builds the power vector (1, t, t^2, t^3), then evaluates the x- and y-components separately.

This lets the same function handle all three curve types. The wrapper functions only provide different control points or tangents and a different geometry matrix.

Generate the Polyline

drawPolySegment samples the curve segment uniformly. With maxLineSegments = 100, this produces 101 points, including the start and end points. Each point is written to the vector in the format expected by drawLines: position x, y, z followed by color r, g, b, a.

Exercise 2: De Casteljau Algorithm

For the second exercise, the solution evaluates the Bezier curve without using a matrix representation. The helper function lerp computes a linear interpolation between two points. computeDeCasteljau repeatedly applies this interpolation to neighboring points until only one point remains.

Draw the Bezier Curve with De Casteljau

drawBezierSegmentDeCasteljau uses the same sampling and output format as drawPolySegment. The only difference is that the curve point for each t is computed with computeDeCasteljau. In the final image, this curve coincides with the matrix-evaluated Bezier curve, but the program draws it slightly lower so the two variants can be compared.

Curve Types in the Main Program

The existing drawing functions for Hermite, Bezier, and B-spline curves keep their structure. Each function sets the appropriate geometry matrix, calls the shared curve evaluation, and also draws the control points. In draw, the curves are only shifted to different y-positions: Hermite at the top, Bezier in the middle, De Casteljau below it, and the B-spline segments at the bottom.
