Triangle Data

Each triangle is stored as a list of three vertices. A vertex consists of position and color, seven floats in the format x y z r g b a. The triangle positions are centered around the local origin, making the local origin the pivot point as well.

This matters for the second subtask: when a triangle rotates around the origin of its own local coordinate system, it automatically rotates around its center. The vertex data does not need to change. All differences between the top, right, and left triangles are created by the model matrices.

From the Assignment to the Model Matrix

The first subtask only requires arranging the three local triangles. A translation is enough for each one: the top triangle is moved upward, and the other two are moved down-left and down-right. The second subtask then adds a rotation around a different axis for each triangle.

Before each draw call, a separate model matrix is set with setDrawTransform. This matrix describes how the local coordinates of the triangle currently being drawn are transformed into world coordinates. drawTriangles then always draws the same kind of local triangle data, but with a different model matrix.

Order of Transformations

In this code, a point is transformed as matrix * vector. Therefore, in a composed matrix, the right-hand matrix acts first: 

p_world = (T * R) * p_local
=  T * (R * p_local)

The matrix farther to the right in the code therefore acts closer to the local object. The matrix farther to the left acts later and describes placement in the scene. You can think of T * R as "rotate the local triangle, then place the result at the Triforce position."

This is why the code uses Mat4::translation(...) * Mat4::rotationX(...). The local point is rotated first. Because the triangle vertices are centered around the local origin, this rotation happens around the triangle center. Only afterward does the translation move the already rotated triangle to its Triforce position.

If the order were reversed and R * T were used, the result would be: 

p_world = (R * T) * p_local
=  R * (T * p_local)

Then the triangle would first be moved away from its local origin and afterward rotated around the scene origin together with that offset. It would no longer stay in its Triforce position, but orbit around the world origin. This is exactly the difference between rotating around an object's own center and rotating around the origin of the whole scene.
