Sheet 12: Transformations

Topic: Matrices and Transformations

In this exercise you work with geometric transformations in two- and three-dimensional space. An object should be translated and rotated around different axes. The important part is not only which transformations are used, but especially the order in which they are multiplied.

The starter code already contains three equilateral triangles with the correct vertex colors. Initially, all triangles lie on top of each other at the origin of their local coordinate system. Use suitable matrices to build an animated Triforce from these three local triangles: one triangle at the top, two below it, and each triangle rotating around its own axis.

In particular, pay attention to the difference between local coordinates and world coordinates. A rotation around the local origin behaves differently from a rotation around the origin of the whole scene. It also matters whether an object is rotated first and translated afterward, or whether these two operations are swapped.

Exercise 1: Arrange the Triforce (10 points)

Compute a separate model matrix for each triangle that moves it to the correct position. The top triangle should move upward, and the other two triangles should move down-left and down-right. Together, the three triangles should form a Triforce.

Use Mat4::translation for this. Before each draw call, use setDrawTransform to set the matrix of the triangle that will then be drawn with drawTriangles.

Make sure each triangle keeps its own local coordinates. Do not modify the vertex data itself; use the matrix to describe where this local triangle should appear in the scene.

Exercise 2: Animate Rotation (10 points)

Extend the program so each triangle rotates by 45 degrees per second around a different major axis:

  • The top triangle rotates around the x-axis.
  • The lower-right triangle rotates around the y-axis.
  • The lower-left triangle rotates around the z-axis.

Compute the angle in animate from the elapsed time. For the rotation, you can use Mat4::rotationX, Mat4::rotationY, and Mat4::rotationZ.

Then combine translation and rotation by matrix multiplication. Each triangle should rotate around its own center while still staying in its Triforce position. Think carefully about whether the rotation must appear before or after the translation in the model matrix.
