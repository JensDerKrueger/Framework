Exercise 1: Barycentric Coordinates

Barycentric coordinates describe a point as a weighted sum of the three triangle vertices. For the given points:

A point lies inside the triangle exactly when all three weights are in the range [0, 1] and their sum is 1. The weights can be computed from the subareas of the triangle. Using signed areas also handles points outside the triangle correctly.

Exercise 2: Ray Through the Pixel

raySphereIntersect receives the ray origin and a point on the image plane. From those two points, the solution first computes a normalized ray direction. It then combines the sphere equation with the ray equation, producing a quadratic equation in the ray distance t.

Discriminant and Intersection Point

The discriminant determines whether the ray hits the sphere. If it is negative, there is no real intersection and the function returns an empty std::optional. Otherwise, the two possible hit distances are computed and the frontmost hit is chosen.

A small epsilon prevents intersections behind the viewer, or exactly at the ray origin, from being accepted as valid hits. For a valid hit, the actual intersection point is computed as rayOrigin + rayDirection * t and returned.

Rendering with the Existing Lighting

The rest of the starter code can remain unchanged: one ray is tested for each pixel. If it hits, the sphere normal is computed from the hit point and the existing Phong lighting code is evaluated. If it misses, the pixel remains black.
