Sheet 11: Texture Coordinates & Sampling

Exercise: Texture coordinates (10 points)

The updated classes from the previous sheet are prepared for texturing support. TODO markers in the code identify the missing components.

Texture coordinates are necessary to map textures correctly onto primitives such as planes, spheres, and triangles. Plane.intersect already computes texture coordinates for hit points. In computer graphics, such coordinates are often visualized as RGB values for debugging, usually with u shown as red and v as green.

The two leftmost images below show a debug view and a scene without textures. You can switch between them with the Spacebar.

For this exercise, implement texture-coordinate computation for spheres using polar coordinates. Sphere accepts three parameters to control the mapping:

  1. Euler angles (Sphere.rotation): rotate the direction vector r around the x-, y-, and z-axis in exactly that order using rotation matrices.
  2. Compute the polar coordinates (phi, theta) and normalize (u, v) to the interval [0, 1].
  3. Scale the texture coordinates with Sphere.scale and then offset them with Sphere.bias.

Exercise: Texture sampling (10 points)

To project textures onto 2D surfaces, implement the corresponding texture-sampling methods.

Familiarize yourself with Texture and the enumerations BorderMode and FilterMode.

Complete the methods marked with TODO so the following combinations are supported:

  • Filters: NEAREST and BILINEAR
  • Border Modes: CLAMP_TO_EDGE, CLAMP_TO_BORDER, MIRRORED_REPEAT

Filter and border modes can be combined freely. The textures in the scene use these combinations deliberately:

  • Earth: REPEAT + NEAREST
  • Light logo: CLAMP_TO_EDGE (everything outside becomes white)
  • Dark logo: MIRRORED_REPEAT in u + CLAMP_TO_BORDER in v, creating a ring effect
  • Checkerboard: REPEAT + NEAREST or BILINEAR

During ray traversal, sampled texture values are automatically multiplied with the ambient and diffuse values of the Phong shading model. You do not need to change these equations.

To test bilinear interpolation, simply change the checkerboard filter mode from NEAREST to BILINEAR.
