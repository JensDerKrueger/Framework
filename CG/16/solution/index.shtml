Exercise 1: Normal Display

For the normal display, one short line is generated per vertex. The start of the line is the vertex position, and its direction is the corresponding vertex normal. The line data stores only the direction; the visible length is applied while drawing through normalLineLength and can be changed with Q and W.

During drawing, the same separation as in the shader is used: the vertex position is a position, and the normal is a direction. Therefore the line start is transformed with the model-view matrix, but the normal direction is transformed with the inverse-transposed model-view matrix. This is deliberately done in C++ before the lines are passed to drawLines.

The simple line shader in the GLApp framework has no concept of normals. For it, the start and end of a line are just positions that would normally both be multiplied by the current model-view transformation. Passing local lines unchanged to that shader would therefore transform the normal direction like a position. The solution first builds finished lines in view space and then sets the identity transform via setDrawTransform(Mat4{}), so the simple shader draws the prepared view-space lines unchanged.

As an additional demonstration, the variable properly controls whether normals are transformed correctly with the inverse-transposed model-view matrix or directly with the model-view matrix. In the running solution, toggle it with P. This is not part of the actual assignment, but it clearly shows how normals and lighting change when the normal transform is wrong.

Exercise 2: Phong-Shader

In the fragment shader, all quantities are in world space. The normal N, the light direction L, and the view direction V are normalized. Ambient light is a constant base fraction of the material color. The diffuse term is the dot product between normal and light direction, clamped to the range from 0 to 1. For the specular term, the incoming light direction is reflected around the normal and compared with the view direction. Since L points from the surface point to the light source, reflect receives the opposite direction, -L; that function expects the incoming light ray.
