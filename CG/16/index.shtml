Sheet 16: Local Lighting

Topic: Local Phong Lighting

In this exercise you complete a small OpenGL scene for local lighting. The program shows a cube and an ellipsoid side by side. Both objects can be rotated around their own centers with the mouse. Alternatively, the point light can be moved around the scene so the influence of light position becomes directly visible.

Starter Program

The starter code already creates the geometry for the cube and ellipsoid with the Tessellation class and draws both objects with separate model matrices. The sphere is scaled non-uniformly so the difference between position and normal transformation becomes visible. The light position is computed from a fixed initial position and its own ArcBall matrix, then passed to the shader in world space.

Exercise 1: Normal Display (15 points)

Complete the normal display in main.cpp. For each vertex, draw a short line that starts at the vertex position and points in the direction of the corresponding vertex normal. Store the data as position and color so it can be drawn with drawLines from the GLApp framework. The length of the displayed normals is controlled by normalLineLength.

When drawing, make sure points and directions are transformed differently.

Exercise 2: Phong Shader (15 points)

Complete direct local Phong lighting in Shader/fragmentShader.frag. The shader already receives the fragment's world position, transformed normal, material color, point light position, and camera position.

The result should consist of three terms:

  • Ambient: a constant base contribution.
  • Diffuse: a Lambert term.
  • Specular: a Phong highlight with reflection vector and exponent.

Make sure all direction vectors are normalized and avoid negative diffuse or specular contributions.

Controls

  • Drag with the left mouse button: perform the current interaction.
  • L: switch between object rotation and light rotation.
  • N: toggle the normal display.
  • ↑ / ↓: zoom in and out.
  • ← / →: move the scene horizontally.
  • Q / W: shorten or lengthen the normal lines.
  • R: reset rotation, zoom, scene shift, and normal length.
