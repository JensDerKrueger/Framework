Exercise 10: Octrees

Topic: Acceleration Structures for Ray Tracing (25 points)

Ray tracing a complex triangle mesh by testing every ray against every object is wasteful. Most rays pass through only a small part of the scene. Acceleration structures exploit this observation: inexpensive bounding volumes are tested first, and only primitives that could actually be hit are visited.

In this exercise you implement an octree. The scene contains a triangle mesh, several spheres, and a large floor. The application already includes a brute-force ray tracer, progressive rendering, a split-screen comparison, and an octree visualization. Your task is to complete the missing octree code in AABB.h and Scene.cpp.

Press M to switch between split-screen comparison, full octree rendering, and full brute-force rendering. Use the up and down arrow keys to zoom. Press B to show or hide octree boxes, and use the left and right arrow keys to change how deeply the hierarchy is drawn. Press R to toggle whether a new ray-traced image starts automatically after the mouse is released. Drag with the left mouse button to rotate the scene.

Provided Ray-Box Helper

The ray-box intersection from the BVH exercise is already available in AABB::intersectInterval. The octree traversal uses this function to determine where a ray enters and exits the root cube. You do not need to implement the slab test again in this exercise.

The AABB function still needed for octree construction is AABB::overlaps. Two boxes overlap exactly when their intervals overlap on all three axes. This is used to decide which child cells are touched by a primitive's bounding box.

Exercise 1: Octree Construction (13 points)

Complete AABB::overlaps, Scene::buildOctree, and Scene::buildOctreeNode. The vector sceneObjects stores the actual objects. The vector octreeObjectIndices stores indices into sceneObjects. The scene objects themselves should not be copied or reordered.

Each octree node stores a cubic bounding box. A leaf node stores a contiguous range in octreeObjectIndices. An inner node stores up to eight child nodes. Stop recursion when a node contains at most OCTREE_LEAF_SIZE objects or when OCTREE_MAX_DEPTH is reached.

Inner nodes split their cube into eight equally sized child boxes. An object is inserted into every child whose box overlaps the object's bounding box. If an object overlaps more than OCTREE_MAX_CHILD_REFERENCES children, keep it in the current node instead. This prevents very large objects from being duplicated throughout the tree.

Construction consists of creating the root and recursively subdividing nodes.

  1. In AABB::overlaps, compare the two boxes on all three axes. The boxes overlap only if no separating axis exists.
  2. In Scene::buildOctree, clear octreeObjectIndices and octreeNodes.
  3. Iterate over sceneObjects. For each object with a non-empty bounding box, add its index to a temporary vector and expand the scene bounding box.
  4. The root of an octree must be a cube. Convert the scene bounding box to a cube with makeCubicBounds, then call buildOctreeNode for the root.
  5. In Scene::buildOctreeNode, create an OctreeNode, store the current node bounds in it, append it to octreeNodes, and remember its index.
  6. If the object count is at most OCTREE_LEAF_SIZE, or OCTREE_MAX_DEPTH has been reached, make the node a leaf. Store the objects as a contiguous range in octreeObjectIndices using firstObject and objectCount.
  7. Otherwise, compute the eight child boxes with computeOctreeChildren. Use one temporary vector per child and one vector for objects that remain in the current node.
  8. For each object, count which child boxes overlap its bounding box. If the number of overlapping children is greater than OCTREE_MAX_CHILD_REFERENCES, put the object into the local object vector of the current node.
  9. If the object overlaps only a few children, add its index to each of those child vectors. The same object can therefore be stored in multiple subtrees.
  10. If all child vectors are empty, subdivision did not help. Store all objects as a leaf range in the current node and return.
  11. Store the local object vector as the current node's own object range. Then recursively build each non-empty child and store the returned node index in the corresponding children entry.

Exercise 2: Octree Traversal (12 points)

Complete Scene::intersectOctree and Scene::intersectOctreeNode. The root function computes the ray interval through the root cube. The node function then traverses octree cells in ray order.

Treat the eight children of an inner node as a 2 x 2 x 2 grid. Start in the child cell that contains the ray entry point. Compute the next cell boundary on each axis and use a 3D DDA to step through the crossed cells. Recurse only into child nodes that the ray actually passes through. Because cells are visited from front to back, you can stop as soon as the nearest hit lies before the next cell boundary.

DDA stands for Digital Differential Analyzer. In this exercise, it means the traversal follows the ray from one child cell to the next instead of recursively testing all eight child boxes. For each axis, store the ray parameter of the next child boundary (tMaxAxis) and the distance in ray-parameter space to the following boundary (tDelta). The smallest tMaxAxis tells you which boundary is crossed first. Then advance the corresponding cell coordinate and add tDelta on that axis.

If the ray crosses an edge or corner of the child grid, two or three tMaxAxis values can be equal. In that case, advance all matching axes. This avoids visiting a cell that the ray only touches numerically, and it also avoids missing the cell behind a grid edge.

Since primitives may be inserted into multiple child cells, use octreeMailbox and octreeMailboxStamp to test each object at most once per ray.

Implement the traversal in this order.

  1. In Scene::intersectOctree, fall back to brute force if no octree exists. Otherwise intersect the ray with the root cube using intersectInterval. If the ray misses the root cube, return no hit.
  2. Prepare the mailbox for this ray. Grow octreeMailbox if necessary, increment octreeMailboxStamp, and clear the mailbox only if the stamp overflows.
  3. Call intersectOctreeNode for the root node with tEnter, tExit, and the current mailbox stamp.
  4. In intersectOctreeNode, first test the objects stored directly in the current node. Check the mailbox before each object test. If the object already has the current stamp, skip it. Otherwise set the stamp and test the object.
  5. Keep the nearest hit and update maxT whenever a closer hit is found. This lets traversal ignore cells and primitives behind the current best hit.
  6. If the node is a leaf, return the best hit among the directly stored objects.
  7. For an inner node, compute the DDA start point as the ray position at max(tEnter, 0), shifted forward slightly by a small epsilon.
  8. Determine the current child cell by comparing this point with the node center. Each coordinate is 0 or 1. Pack the child index as x + 2*y + 4*z.
  9. For each axis, compute step, tMaxAxis, and tDelta. tMaxAxis is the ray parameter of the next child boundary on that axis; tDelta is the parameter distance between child boundaries.
  10. Visit child cells while the cell coordinate is inside the 2 x 2 x 2 grid and the current interval still lies before tExit and maxT.
  11. Recurse into a visited cell only if the corresponding child node exists. Pass the interval [currentT, nextT] for that child cell.
  12. Because cells are visited front to back, stop immediately if the current nearest hit lies inside the current child interval.
  13. Advance to the next child cell by moving every axis whose tMaxAxis equals the next boundary. Advancing multiple axes is important when the ray crosses an edge or corner between cells.
