Exercise 14: Triangle Rasterization

Topic: Edge Functions and Bounding Boxes (25 points)

In the software rasterization exercises for diffuse and Phong lighting, triangles were drawn by testing every pixel of the whole image. That is simple, but wasteful. This exercise keeps the basic idea of shading pixels inside a triangle, but replaces the brute-force loop over the whole image with a GPU-like triangle rasterizer.

The central tools are a screen-space bounding box and three edge functions. The bounding box restricts the set of tested pixels. The edge functions decide whether a pixel center lies inside the triangle and also provide barycentric weights for interpolation.

Exercise 1: Edge Function (5 points)

Implement edgeFunction in main.cpp. The function returns the signed area expression for the directed edge a -> b and the point p = (x, y). The sign indicates which side of the edge the point lies on. Use the formula above directly. The function should not perform the inside test itself; it should only return the signed value.

Exercise 2: Rasterization with a Bounding Box (7 points)

Complete the first part of drawTriangle. Compute the signed area, discard degenerate triangles, handle both orientations, and compute an integer bounding box clamped to the image size. The rasterizer should visit only pixels inside this box.

The signed triangle area can be computed with an edge function, for example edgeFunction(b, c, a.x, a.y). This is the code version of the area expression shown above. If this area is negative, the triangle has the opposite orientation. Store an orientation factor 1 or -1 and multiply each edge value by it. This keeps the later inside test independent of vertex order.
