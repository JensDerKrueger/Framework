DDA

DDA follows the continuous line with floating-point increments. The dominant axis determines how many steps are needed. Each floating-point position is rounded to the nearest pixel.

The implementation computes steps = max(abs(dx), abs(dy)). This advances the dominant axis by exactly one pixel per iteration, while the other axis changes by a fraction. The algorithm is short and readable, but it requires floating-point arithmetic and rounding in the loop.

Bresenham

Bresenham replaces floating-point steps with an integer error term. This error term decides whether the next pixel advances horizontally, vertically, or diagonally.

The all-octants variant stores the positive distances dx and dy as well as the signs sx and sy. The doubled error value is used to make the next pixel decision with integer comparisons. A step in x subtracts dy from the error; a step in y adds dx.
