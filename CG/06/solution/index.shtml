Exercise 1: Phong Shader State

PhongShader stores all values needed for local per-pixel lighting: the positions of the light and viewer, ambient, diffuse, and specular light color, and the exponent for the specular highlight. The copy constructor copies the same values so BumpPhongShader can reuse the existing Phong shader.

Per-Pixel Phong Lighting

shade receives the interpolated surface point from triangle rasterization. First, the ambient term is computed from material color and light color. Then the normal and light direction are normalized, and the diffuse term is determined with max(0, dot(N, L)).

For the specular term, the solution computes the reflected light direction R and the view direction V. The clamped dot product dot(V, R) is raised to the stored exponent and multiplied by the specular material and light colors. The sum of ambient, diffuse, and specular terms is the final color.

Exercise 2: Prepare the Bump Shader

BumpPhongShader inherits from PhongShader. After modifying the normal, it can therefore use the same Phong lighting code. It additionally stores cellSize for scaling the x-y position and bumpHeight for the height of the procedural bumps.

Procedural Height Function

The helper functions f, fx, and fy implement the sine function from the assignment and its partial derivatives. f returns the bump height, while fx and fy describe the local slope in x- and y-direction.

Replace the Normal and Apply Phong

In the shade call, the pixel position is first divided by cellSize so the bump function repeats at the desired scale. The z-position is set to bumpHeight * f(x, y).

For a height field z = f(x, y), the normal points in the direction (-df/dx, -df/dy, 1). The solution therefore replaces the interpolated normal with the bump normal computed from the derivatives and then calls PhongShader::shade. The lighting computation itself remains unchanged.
