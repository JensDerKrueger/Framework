Exercise 09: Bounding Volume Hierarchies

Topic: Acceleration Structures for Ray Tracing (25 points)

Ray tracing a complex triangle mesh by testing every ray against every object is wasteful. Most rays pass through only a small part of the scene. Acceleration structures exploit this observation: inexpensive bounding volumes are tested first, and only primitives that could actually be hit are visited.

In this exercise you implement a Bounding Volume Hierarchy (BVH). The scene contains a triangle mesh, several spheres, and a large floor. The application already includes a brute-force ray tracer, progressive rendering, a split-screen comparison, and a BVH visualization. Your task is to complete the missing BVH code in AABB.h and Scene.cpp.

Press M to switch between split-screen comparison, full BVH rendering, and full brute-force rendering. Use the up and down arrow keys to zoom. Press B to show or hide BVH boxes, and use the left and right arrow keys to change how deeply the hierarchy is drawn. Press R to toggle whether a new ray-traced image starts automatically after the mouse is released. Drag with the left mouse button to rotate the scene.

Exercise 1: Ray-Box Intersection (7 points)

Complete AABB::intersectInterval. Use the slab method: for each coordinate axis, compute the interval in which the ray lies between the two planes of the box. The intersection of the three intervals is the part of the ray that runs through the box.

The function receives tMin and tMax. Treat these as the currently valid ray interval. Store the final interval in tEnter and tExit. The BVH traversal only needs the boolean wrapper AABB::intersect, but the interval computation is the same ray-box test used in the octree exercise.

Remember the parallel-ray case. If a ray is parallel to a pair of slab planes, it intersects the box only if its origin lies between those two planes.

  1. If the box is empty, return false.
  2. Read the ray origin and direction once before the loop.
  3. Iterate over the three coordinate axes and intersect the ray with the two planes of the corresponding slab.
  4. If the ray direction on this axis is almost zero, treat it as a parallel ray. The ray can hit the box only if its origin coordinate lies between minCorner and maxCorner.
  5. For a non-parallel ray direction, compute the two intersection parameters t0 and t1. Swap them if necessary so t0 is the nearer plane and t1 the farther plane.
  6. Intersect this slab interval with the current valid ray interval: tMin = max(tMin, t0) and tMax = min(tMax, t1).
  7. If the interval becomes empty, return false. If all axes survive, write the final interval to tEnter and tExit and return true.

Exercise 2: BVH Construction (10 points)

Complete Scene::buildBVH and Scene::buildBVHNode. The vector sceneObjects stores the actual objects. The vector bvhObjectIndices stores indices into sceneObjects and may be freely reordered while the hierarchy is built.

Each BVH node stores a bounding box. A leaf node stores a contiguous range in bvhObjectIndices. An inner node stores the indices of its two child nodes. Stop recursion when a node contains at most BVH_LEAF_SIZE objects.

Inner nodes are split at the median of the bounding-box centers. Use the axis along which the centers have the largest extent. std::nth_element can partition the range without fully sorting it.

Unlike an octree, the BVH does not subdivide space into fixed cells. It subdivides an object list. Each object appears exactly once in the hierarchy, and each leaf stores a range of indices in bvhObjectIndices.

Build the hierarchy in this order.

  1. In Scene::buildBVH, clear bvhObjectIndices and bvhNodes.
  2. Iterate over sceneObjects. For each object with a non-empty bounding box, append its index to bvhObjectIndices. Do not copy or reorder sceneObjects itself.
  3. If bvhObjectIndices is not empty, call buildBVHNode(0, bvhObjectIndices.size()). The returned node is the root. In this implementation the root is at index 0 because it is appended first.
  4. In Scene::buildBVHNode, create a node for the half-open range [begin, end) in bvhObjectIndices.
  5. Compute the node's bounding box by uniting the bounds of all objects in that range. At the same time, compute a second bounding box over the object centers; this center box is used only to choose the split axis.
  6. Append the new BVHNode to bvhNodes and remember its index. The child indices are filled after the recursive calls.
  7. If the range contains at most BVH_LEAF_SIZE objects, make the node a leaf: store firstObject = begin and objectCount = end - begin, then return the node index.
  8. Otherwise, choose the split axis from the extent of the center box. Use the axis with the largest extent because that is where the object centers are spread out the most.
  9. Compute middle = begin + objectCount / 2. Use std::nth_element to partition bvhObjectIndices so objects before middle have smaller center coordinates on the split axis than objects after middle. A full sort is unnecessary.
  10. Recursively build the left child over [begin, middle) and the right child over [middle, end). Store the returned node indices in left and right, then return the current node index.

Exercise 3: BVH Traversal (8 points)

Complete Scene::intersectBVHNode. First test the ray against the bounding box of the current node. If the box is missed, the whole subtree can be skipped.

If the node is a leaf, intersect the ray with all objects in the stored leaf range and return the nearest hit. If the node is an inner node, traverse both children and return the nearest child hit. As soon as a nearer hit is found, use its distance to shrink maxT. Later box and primitive tests can then ignore farther intersections.

Implement the traversal in this order.

  1. In Scene::intersectBVH, fall back to brute force if no BVH exists. Otherwise call intersectBVHNode for node 0 with maxT = infinity.
  2. In Scene::intersectBVHNode, read the current node from bvhNodes.
  3. Test the ray against the node bounding box with AABB::intersect. Pass 0 as the lower interval bound and the current maxT as the upper bound. If the box is missed, return no hit.
  4. If the node is a leaf, iterate over node.objectCount objects. The actual object index is stored in bvhObjectIndices[node.firstObject + i].
  5. Intersect the ray with each object in the leaf. Ignore misses and hits with t > maxT. Keep the nearest hit found so far.
  6. When a nearer leaf hit is found, update maxT to that hit distance. This lets the remaining primitive tests ignore farther intersections.
  7. If the node is an inner node, traverse the left child first. If the left child reports a hit, shrink maxT to its distance.
  8. Traverse the right child with the reduced maxT. If the right child returns a hit, it is nearer than the left hit because the right subtree was limited to the left hit distance. Return the right hit; otherwise return the left one.
  9. This traversal is intentionally simple. A more advanced implementation could visit children in ray order, but that is not required for this exercise.

Questions

  1. Why does the BVH help much more for the bunny mesh than for a scene with only a few spheres?
  2. What happens when BVH_LEAF_SIZE is chosen very small or very large?
  3. Why can a poor hierarchy be slower than brute force for a tiny scene?
  4. What does the BVH overlay reveal about the chosen split strategy?
