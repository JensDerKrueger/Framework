Sheet 13: Image Compression

Exercise: A Minimal JPEG Encoder (30 points)

In this exercise you implement the central lossy stages of a small baseline JPEG encoder. The project already provides the OpenGL display, JPEG marker writing, Huffman tables, bit packing, run-length coding, and file output. Your code fills in the image-processing steps that make JPEG compression work.

The application shows the original image on the left and the reconstructed compressed image on the right. Use Up and Down to change the quality factor, C to cycle chroma subsampling, and M to cycle the quantization matrix. In the native application, W writes the current compressed image as a valid JPEG file.

Task 1: RGB to YCbCr (6 points)

Complete convertToYCbCr in JPEGCodec.cpp. JPEG does not directly compress RGB values. It first separates brightness from color differences: Y stores luminance, while Cb and Cr store blue-yellow and red-cyan color differences around the neutral value 128.

Use the formulas given in the source comments. Pixels outside the original image should repeat the closest border pixel so that the padded image dimensions can be encoded in complete JPEG macroblocks.

Task 2: Chroma Subsampling (7 points)

Complete downsampleChroma422 and downsampleChroma420. Chroma subsampling reduces the resolution of the color-difference channels while keeping the luminance image unchanged. This is useful because the human visual system is usually more sensitive to luminance detail than to chroma detail.

In 4:2:2, average two horizontal chroma samples. In 4:2:0, average a 2 x 2 chroma block. Use C in the running program to compare 4:4:4, 4:2:2, and 4:2:0.

Task 3: 8 x 8 DCT (10 points)

Complete computeDCTBlock. JPEG transforms each 8 x 8 image block from spatial samples into frequency coefficients. The first coefficient is the block average. The other coefficients describe horizontal, vertical, and diagonal oscillations of increasing frequency.

Before the transform, shift each sample by subtracting 128. Then compute all 64 DCT coefficients using the formula in the source comments. The cosine terms are already precomputed in dctBasis().

Task 4: Quantization (7 points)

Complete quantizeBlock. Divide each DCT coefficient by the corresponding quantization-table entry and round the result to an integer. This is the main lossy step in baseline JPEG: larger table values preserve less detail and produce more zero coefficients.

Use M to cycle through different quantization matrix presets. Compare how the image quality and compressed byte count react to the quality factor and to the selected matrix. The quality factor scales the quantization matrix: higher quality means smaller quantization values, fewer discarded coefficients, and a larger file.
