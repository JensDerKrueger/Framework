Task 1: Diffuse Sampling

The diffuse bounce uses the same cosine-weighted construction described in the exercise text: add a random unit vector to the surface normal and normalize the result. This creates more samples near the normal than near grazing directions, which matches the cosine factor of Lambertian reflection.

Task 2: Trace One Path

The path tracer carries a ray and a throughput value. A miss means the path reached the environment light. An emissive material terminates the path immediately; in this simplified renderer a material either emits light or scatters the path, but not both. 

Loading Scene.cpp...

Refractive and mirror materials reuse the raytracer's deterministic reflection and refraction operations. The difference is that the path tracer chooses only one continuation direction. For glass, the Fresnel term is used as the probability of reflection; otherwise the path refracts.

After choosing the next direction, the new ray starts slightly away from the surface to avoid self-intersection. If the loop reaches maxDepth before finding environment or emissive light, this simple implementation returns black.

Behavior Questions

The first scene is faster per ray because many paths leave the scene quickly and hit the environment after only a few intersections. The Cornell box is closed, so paths tend to bounce more often before they terminate. More bounces mean more intersection tests and more material decisions per pixel sample.

The first scene also converges faster because the environment light is easy to find: many randomly chosen paths collect useful light. In the Cornell box, the important light source is harder for random paths to hit, and indirect lighting depends on more bounces. That increases variance, so the image needs many more samples before the noise disappears.
