Sheet 2: Bounding Volume Hierarchies

Exercise: Acceleration Structures for Ray Tracing (25 points)

Ray tracing a complex triangle mesh by testing every ray against every object is wasteful. Most rays only pass through a small part of the scene. Acceleration structures exploit this observation by first testing cheap bounding volumes and only visiting the primitives that may actually be hit.

In this exercise you will implement a bounding volume hierarchy (BVH). The scene contains a triangle mesh, several spheres, and a large floor. The application already contains a brute-force ray tracer, progressive rendering, a split-screen comparison mode, and a BVH visualization mode. Your task is to complete the missing BVH code in AABB.h and Scene.cpp.

Use M to switch between split-screen comparison, full BVH rendering, and full brute-force rendering. Use Arrow Up/Arrow Down to zoom. Use B to toggle the BVH box overlay, and Arrow Left/Arrow Right to change how deep into the hierarchy the overlay is drawn. Use R to toggle whether a new ray-traced image starts automatically after releasing the mouse. Drag with the left mouse button to rotate the scene.

Task 1: Ray-Box Intersection (7 points)

Complete AABB::intersectInterval. Use the slab method: for each coordinate axis, compute the interval in which the ray lies between the two planes of the box. The intersection of the three intervals is the part of the ray inside the box.

The function receives tMin and tMax. Treat these as the current valid ray interval. Store the final interval in tEnter and tExit. The BVH traversal only needs the boolean wrapper AABB::intersect, but computing the interval is the same ray-box test used in the octree exercise.

Remember the parallel-ray case. If a ray is parallel to one pair of slab planes, it only intersects the box if its origin lies between those two planes.

  1. If the box is empty, return false.
  2. Read the ray origin and direction once before the loop.
  3. Loop over the three coordinate axes. For each axis, intersect the ray with the two planes of the corresponding slab.
  4. If the ray direction on this axis is almost zero, handle it as a parallel-ray case. The ray can only hit the box if its origin coordinate lies between minCorner and maxCorner on this axis.
  5. For a non-parallel ray direction, compute the two intersection parameters t0 and t1. Swap them if necessary so that t0 is the near plane and t1 is the far plane.
  6. Intersect this slab interval with the current valid ray interval: tMin = max(tMin, t0) and tMax = min(tMax, t1).
  7. If the interval becomes empty, return false. If all axes survive, write the final interval to tEnter and tExit, then return true.

Task 2: BVH Construction (10 points)

Complete Scene::buildBVH and Scene::buildBVHNode. The vector sceneObjects stores the actual objects. The vector bvhObjectIndices stores indices into sceneObjects and may be reordered freely while building the hierarchy.

Each BVH node stores a bounding box. A leaf node stores a contiguous range in bvhObjectIndices. An inner node stores the indices of its two child nodes. Stop recursion when a node contains at most BVH_LEAF_SIZE objects.

For inner nodes, split the objects by the median of their bounding-box centers. Use the axis where the centers have the largest extent. You can use std::nth_element to partition the range without fully sorting it.

Unlike the octree, the BVH does not subdivide space into fixed cells. It subdivides a list of objects. Every object appears exactly once in the hierarchy, and every leaf stores a range of indices into bvhObjectIndices.

Build the hierarchy in this order.

  1. In Scene::buildBVH, clear bvhObjectIndices and bvhNodes.
  2. Iterate over sceneObjects. For each object with a non-empty bounding box, append its index to bvhObjectIndices. Do not copy or reorder sceneObjects itself.
  3. If bvhObjectIndices is not empty, call buildBVHNode(0, bvhObjectIndices.size()). The returned node becomes the root. In this implementation the root is stored at index 0 because it is the first node appended.
  4. In Scene::buildBVHNode, build one node for the half-open range [begin, end) inside bvhObjectIndices.
  5. Compute the node bounding box by joining the bounds of all objects in this range. At the same time, compute a second bounding box over the object centers. This center box is only used to choose a split axis.
  6. Append the new BVHNode to bvhNodes and remember its index. Child indices will be filled in after the recursive calls.
  7. If the range contains at most BVH_LEAF_SIZE objects, make this node a leaf by storing firstObject = begin and objectCount = end - begin, then return the node index.
  8. Otherwise, choose the split axis from the center-bounds extent. Use the axis with the largest extent, because this is where the object centers are spread out most.
  9. Compute middle = begin + objectCount / 2. Use std::nth_element to partition bvhObjectIndices so that objects before middle have smaller center coordinates on the split axis than objects after middle. A full sort is not necessary.
  10. Recursively build the left child over [begin, middle) and the right child over [middle, end). Store the returned node indices in left and right, then return the current node index.

Task 3: BVH Traversal (8 points)

Complete Scene::intersectBVHNode. First test the ray against the current node's bounding box. If the box is missed, the whole subtree can be skipped.

If the node is a leaf, intersect the ray with all objects in the stored leaf range and return the closest hit. If the node is an inner node, traverse both children and return the closest child hit. Whenever a closer hit is found, use its distance to tighten maxT. This allows later box and primitive tests to ignore intersections that are farther away.

Implement the traversal in this order.

  1. In Scene::intersectBVH, return to brute force if no BVH exists. Otherwise call intersectBVHNode for node 0 with maxT = infinity.
  2. In Scene::intersectBVHNode, fetch the current node from bvhNodes.
  3. Test the ray against the node's bounding box using AABB::intersect. Pass 0 as the lower ray interval and the current maxT as the upper interval. If the box is missed, return no hit.
  4. If the node is a leaf, loop over node.objectCount objects. The actual object index is stored at bvhObjectIndices[node.firstObject + i].
  5. Intersect the ray with each object in the leaf. Ignore missed objects and hits with t > maxT. Keep the closest hit found so far.
  6. Whenever a closer leaf hit is found, update maxT to that hit distance. This lets the remaining primitive tests ignore farther intersections.
  7. If the node is an inner node, recursively traverse the left child first. If the left child reports a hit, tighten maxT to the left hit distance.
  8. Traverse the right child with the tightened maxT. If the right child returns a hit, it is closer than the left hit because the right subtree was clipped to the left hit distance. Return the right hit. Otherwise return the left hit.
  9. This traversal is intentionally simple. A more advanced implementation could visit children in ray order, but that is not required for this exercise.
