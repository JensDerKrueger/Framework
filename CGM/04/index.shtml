Sheet 4: Growth Simulation & Spatial Subdivision

Topic: Growth Simulation

The provided code simulates the growth of a dendrite-like structure using particles. The main logic is implemented in the simulate function in main.cpp.

First, a seed particle is placed at the origin. In every subsequent step, a new particle is generated at a random position in the range from (-1, -1, -1) to (1, 1, 1). It is then moved in random directions until it collides with an existing particle. If it moves too far away from the origin, a new random position is generated.

Without an acceleration structure, each new particle would have to be checked against all existing particles. To speed this up, the code uses an octree for spatial subdivision. The current octree structure can be visualized directly.

You can switch between a 2D and 3D visualization by commenting or uncommenting the second line in main.cpp.

Exercise: Octree as an Acceleration Structure (20 points)

The Octree class consists of a hierarchical structure of OctreeNode objects. The root attribute points to the root of the tree.

Two central parameters control the structure:

  • maxElemCount: maximum number of elements per node before a split occurs
  • maxDepth: maximum depth of the octree

The provided framework already contains a nearest-neighbor search in Octree::minDist to determine the minimum distance to the current particle position.

Your task is to correctly implement the insertion of new elements into the octree and to create child nodes whenever subdivision becomes necessary.

Complete the following methods in the OctreeNode class:

  • OctreeNode::add
  • OctreeNode::split

Both methods are marked with TODO comments in the source code. These comments provide hints for the concrete implementation.

For visual validation, compare your result with the reference solution on this page. Note that the growth structure is based on random placement, so every simulation run produces slightly different results.
