Sheet 11: Screen Space Ambient Occlusion

Exercise: SSAO in a Deferred Renderer (25 points)

In the previous deferred shading exercises the renderer first filled a G-buffer and then computed lighting in a full-screen image-space pass. This exercise adds screen space ambient occlusion, usually abbreviated as SSAO. SSAO estimates how much of the local hemisphere around a visible surface point is blocked by nearby visible geometry. The result is not a physically complete global illumination solution, but it gives useful contact darkening in creases, intersections, and dense object arrangements.

The scene, G-buffer pass, deferred lighting pass, and debug views are already present. Your task is to add two additional full-screen passes. The first pass writes a raw ambient occlusion texture. This raw estimate is intentionally somewhat noisy because it uses only a small number of samples. The next two passes smooth this texture with a separable blur: one horizontal pass and one vertical pass. The final deferred shader then reads the smoothed result and uses it to darken the ambient term and local contact regions.

Task 1: Raw Ambient Occlusion Pass (6 points)

Complete renderScreenSpaceAmbientOcclusion in main.cpp and call it from the raw AO pass in draw. This pass is very similar to the final deferred lighting pass: it disables depth testing, activates a full-screen shader, binds G-buffer textures, and draws one full-screen triangle into rawAmbientOcclusionBuffer.

  1. Pass projectionMatrix to the SSAO shader.
  2. Pass the parameters radius, bias, intensity, sampleCount, and maxMipLevel.
  3. Bind normalBuffer and positionBuffer.
  4. Draw the full-screen triangle into ambientOcclusionBuffer.

Good starting values are radius = 16.0, bias = 0.60, intensity = 2.10, sampleCount = 24, and maxMipLevel = 5.0. The AO texture is cleared to white before this pass. White means no occlusion, so the program should still render normally while the pass is incomplete.

Task 2: Estimate SSAO in the Shader (10 points)

Complete estimateAmbientOcclusion in Shader/ssao.frag. For each visible pixel, read its view-space position and normal from the G-buffer. Use the provided sample directions as a small set of points in the hemisphere above the surface.

Each sample direction is transformed into view space, scaled by the SSAO radius, and added to the current position. The resulting sample position is projected back to screen coordinates with projectionMatrix. At that projected screen location, read the position buffer. If the stored position is closer to the camera than the projected sample position, nearby geometry blocks that sample direction and contributes to occlusion.

Use textureLod for the position lookup and choose coarser mip levels for larger sample radii. This approximates a larger support area without requiring many more samples. At the end, convert the average occlusion into an accessibility value: 1 means open and 0 means strongly occluded.

Task 3: Smooth the AO Texture (5 points)

Complete renderAmbientOcclusionBlurPass in main.cpp and Shader/ssaoBlur.frag. The raw SSAO image is useful as an intermediate result, but it is noisy. Separate smoothing passes are the main reason we keep SSAO as its own image-space texture instead of computing it directly inside the final lighting shader.

Use a separable blur. First blur horizontally from rawAmbientOcclusionBuffer into a temporary texture. Then blur vertically from the temporary texture into ambientOcclusionBuffer. Each pass uses only five samples, so the two-pass version needs 10 samples per pixel instead of 25 for a direct 5 x 5 blur. Make the blur edge-aware by also reading positionBuffer: samples with very different view-space depth should receive less weight.

Task 4: Use SSAO in Deferred Lighting (4 points)

Complete the SSAO-related code in Shader/deferred.frag. Read the ambient occlusion texture at the current screen coordinate. This texture now contains the smoothed AO result, not the raw noisy estimate. The value should darken the ambient term because locally hidden regions receive less environment light.

In this teaching example we also use the SSAO value as a soft contact term for direct Phong light. This is not a full physical lighting model, but it makes the screen-space occlusion visible in the final image. Press A to inspect the raw AO buffer and D to inspect the G-buffer.
