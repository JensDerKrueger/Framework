Sheet 12: Sampling

Exercise: Supersampling And Jittered Sampling (25 points)

A rendered image is a finite set of samples of a continuous image signal. If the signal contains details that are too small for the pixel spacing, point sampling produces aliases: jagged edges, lost detail, and structured false patterns. In this exercise you implement several simple anti-aliasing strategies and compare them on the same analytic image signal.

The project already contains the OpenGL window, image display code, and a continuous test signal. The program renders four images. The top-left image is point sampling, the top-right image is regular 4 x 4 supersampling, the bottom-left image uses jittered stratified 8 x 8 sampling, and the bottom-right image is a higher-quality jittered 16 x 16 reference.

Task 1: Regular Supersampling (8 points)

Complete regularSampleOffset in main.cpp. The function receives the integer coordinate of a sub-cell inside an N x N sampling grid. It should return the center of that sub-cell as a sub-pixel offset in [0, 1]^2.

For example, in a 4 x 4 grid, the first sample should be located at (0.125, 0.125), not at the pixel corner. The averaging loop in samplePixel already evaluates all sub-samples and divides by the total weight.

Task 2: Sample Count Per Mode (6 points)

Complete samplesPerAxisForMode. Regular supersampling and jittered supersampling should use different sample counts here, so the image quality improves in reading order: point sampling first, then regular supersampling, then jittered supersampling, and finally the reference image.

Use 4 x 4 samples for regular supersampling, 8 x 8 samples for jittered sampling, and 16 x 16 jittered samples for the reference. The reference is more expensive, but it gives a better approximation of the pixel average and serves as a target image for the comparison.

Task 3: Jittered Stratified Sampling (9 points)

Complete jitteredSampleOffset. Regular supersampling still repeats the same sample pattern in every pixel, which can leave visible aliases. Jittered stratified sampling divides the pixel into the same regular sub-cells but places one random sample inside each sub-cell.

Use the built-in Vec2::random() helper to generate two random numbers in [0, 1). These numbers should move the sample inside its own stratum, not across the whole pixel.

Task 4: Compare The Results (2 points)

Compare the four images. Point sampling should show strong aliasing. Regular supersampling should reduce the problem but can still create structured patterns. Jittered sampling should turn some of those structured aliases into less regular noise. The reference image should show the least aliasing because it uses more samples. Press S to switch between test signals and use the arrow keys to change the signal frequency.
