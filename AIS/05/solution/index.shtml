Solution Files

The provided files extend the texturing solution with shadow mapping. A depth texture is rendered from the light's point of view and then sampled during the normal-mapped lighting pass for the ground plane and teapot.

Shadow Map Resources

Shadow rendering uses a small depth-only shader program, a framebuffer, and a GLDepthTexture. The clip-to-texture matrix maps homogeneous clip-space coordinates into texture lookup coordinates for the later shadow test.

Light-Space Matrices

Each frame builds the camera view matrix, the animated light transform, the light projection, and a light view matrix that looks from the light position toward the origin. Their product is stored as worldToShadowMatrix.

Two-Pass Rendering

The same scene function is used twice. In the first pass it writes only depth values into the shadow map from the light's perspective. In the second pass it renders the real scene from the camera and binds the completed shadow map as a texture.

Passing Shadow Coordinates

The vertex shader computes shadowPos by transforming each model-space vertex into light texture space. The fragment shader can then compare the current fragment against the stored depth value.

Applying the Shadow

The fragment shaders use sampler2DShadow and textureProj for the depth comparison. A small bias reduces self-shadow artifacts. Lit fragments receive the full Phong result, while shadowed fragments keep only the ambient contribution.
