Vertex Colors in the Shaders

The vertex shader receives a position and a color for each vertex. The position is transformed by the projection and model-view matrices. The color is passed to the fragment shader as an interpolated varying, so fragments inside a triangle receive smoothly blended colors.

Triangle Geometry and Colors

The solution stores one equilateral triangle in object space and reuses it three times. A separate color array stores three RGB colors for each triangle. The three vertex arrays select the same position buffer but different offsets into the color buffer.

There are many valid ways to solve this assignment, and several of them are shorter, faster, or more elegant than the version shown here. This solution intentionally uses a very basic approach: separate vertex array objects describe the three triangles, each with the same coordinates but different colors. Alternatives include using a single buffer with different offsets for each draw call, or using instancing.

Buffers and Vertex Arrays

Two buffers are created: one for positions and one for colors. The three vertex array objects describe how to read these buffers for the top, right, and left triangle. The color attribute offset changes for each VAO, which gives each triangle its own color triplet while reusing the same triangle positions.

Transformations and Animation

The animation angle is computed from elapsed time at 45 degrees per second. Each triangle uses a different model-view matrix: the top triangle rotates around the x-axis, the right triangle around the y-axis, and the left triangle around the z-axis. Extra translations move the local origin to the requested pivot before rotation and move it back afterward.
