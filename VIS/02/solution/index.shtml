Grayscale Conversion

The grayscale function first checks whether the image has at least three color components. It then chooses either uniform weights or the common luminance weights 0.299, 0.587, and 0.114 for red, green, and blue.

Each pixel is converted by reading its RGB values, computing the weighted sum, and writing the same value back to the image.

ASCII Luminance

The helper getLumiValue converts the current pixel format to one brightness value. Single-channel images can be used directly, two-channel images are averaged, and RGB or RGBA images use the same weighted luminance formula as the non-uniform grayscale mode.

ASCII Art Output

The ASCII conversion maps each luminance value to a character in the selected lookup table. Dark pixels select characters near the beginning of the table, bright pixels select characters near the end. Each character is written twice to compensate for the typical aspect ratio of monospace text.

Image Filtering

The filter function keeps a copy of the input image so every convolution reads from the unchanged source data. It skips the outer border according to the kernel radius, applies the kernel independently to each image component, clamps the result to the valid byte range, and stores it in the temporary image.

Keyboard Shortcuts

The provided key bindings call the implemented operations directly: G and H trigger weighted and uniform grayscale, M applies a mean filter, A and B apply the two Sobel kernels, R reloads the image, and T writes the ASCII output to ascii.txt.
