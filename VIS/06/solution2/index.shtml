Building Integral Curves

The vector version of advect creates one line at a time. Each line starts at a random seed position, and every following point is generated from the previous point using the same Euler-style advection idea from the particle exercise. If a line terminates early, the solution stores the previous point again as a duplicate endpoint. When a new point is identical to the previous one, the line has reached the boundary condition and generation for that line stops.

Boundary Handling

The single-point advection function checks whether the current point is still inside the unit cube. Unlike the particle tracer, it does not reset invalid points to a new random position. Instead, it returns the unchanged point, which lets the line builder create the duplicate endpoint used as the stop condition.

Valid points are advanced with one explicit Euler step: position + flow.interpolate(position) * deltaT.

Line Render Data

The computed line points are converted into a line-list buffer. Every segment needs two endpoints, and each endpoint stores three OpenGL-space coordinates plus RGBA color. As in the particle exercise, the original [0, 1] position is reused as color.

Before a segment is emitted, the render conversion checks whether two consecutive points are equal. This is why an early-terminated line needs the duplicate point: it marks the end of the line without drawing an extra segment.

Drawing and Interaction

The draw function renders the static integral curves as a line list using the current arcball rotation. Mouse dragging updates that rotation, so the generated 3D flow structure can be inspected interactively.
