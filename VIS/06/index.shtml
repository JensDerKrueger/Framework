Exercise: Particles (10 points)

Flowfield visualized using (animated) particles.

Do not change any helper classes in the provided Utils folder!

Before beginning, make yourself familiar with the Flowfield class.

In this task, you will implement the advection of particles in a three-dimensional flow field. Complete the advect function in main.cpp.

Each particle position is stored in the normalized flow-field domain [0, 1]^3. Use an explicit Euler step to move a particle from its current position p to the next position: p_next = p + flow.interpolate(p) * deltaT.

If a particle moves out of bounds, meaning any component is smaller than 0 or larger than 1, reset it to a new random position. The resulting moving particle cloud should follow the flow field over time and make the structure of the vector field visually apparent.

Exercise: Lines (10 points)

Flowfield visualized using lines.

Flow fields can be visualized not only by moving particles, but also by lines that follow the local flow direction. In this task, extend your previous particle-based solution to produce static flow-aligned lines. Complete the initLines() function in main.cpp.

You may reuse and modify your previous advection implementation. Start each line at a random seed position in the normalized domain [0, 1]^3. Then repeatedly advance the previous point with an explicit Euler step using flow.interpolate. The resulting line should appear static and clearly show the directional structure of the flow field.

Unlike the particle exercise, a line should not reset to a new random position when it leaves the domain. Instead, terminate the line. The provided render conversion stops a line when it finds two consecutive identical points. Therefore, if a line terminates early, store a duplicate of the last valid point as the next point. This duplicate endpoint acts as a sentinel and prevents a segment from being drawn outside the volume.
