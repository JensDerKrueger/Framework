Initialization

The app initializes OpenGL state, sets a transparent black background, and creates a random set of particles in the unit cube. The render buffer stores seven floats per particle: three position values followed by four color values.

Particle Advection

The exercise asks for particle advection with the Euler method. Each animation step replaces every particle position with the result of the single-particle advection function.

Particles that have left the unit cube, meaning any component is smaller than 0 or larger than 1, are respawned at a random position. Valid particles move by the interpolated flow vector scaled by the elapsed time, so the particle cloud follows the flow field and makes its structure visible.

Render Data

Particle positions live in the flow-field coordinate range [0, 1]. For rendering, they are converted to OpenGL coordinates in [-1, 1]. The original position is reused as the RGB color, giving the particle cloud a spatial color gradient.

Animation and Drawing

The animation callback computes the time since the previous frame, advances the particles by that amount, and updates the render buffer. The draw function then applies the camera and arcball rotation before drawing the particle buffer as points.

Interaction

Mouse dragging updates the arcball rotation used during drawing. Pressing I calls initParticles again, which resets the particle positions to a new random distribution.
