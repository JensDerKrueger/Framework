Getting started

Please make sure to upload your solutions to Moodle before the submission deadline. Submissions not uploaded before the deadline will not be graded (0 points). Solutions that do not compile will also receive 0 points. Sample input and output are often provided so you can test your programs; solutions that do not correctly handle these examples will only receive partial points. Please do not add any extra output to your program unless explicitly requested. Submit your solution as a single zip file named as follows: a1_Firstname_Lastname.zip.

Compress all files you have modified into one archive. If the exercises include theoretical parts, please also include a text file, scan, or photo with your answers.

Assignments consist of theoretical and programming tasks. The programming tasks require a C++ environment and downloading the exercise code.

Setup the Build Environment

In order to get you started for the assignments, please follow these steps to setup your programming environment. Not all of these steps are mandatory for the first assignment sheets but they will become important once we start with assignments that require OpenGL. Getting used to them now will save you trouble for later assignments.

For Windows users:
  1. You should download the latest Visual Studio IDE (NOT Visual Studio Code). This can be downloaded for free from Microsoft. The Community Edition is sufficient for our needs.
  2. Run the installer and check 'Desktop development with C++' in the 'Desktop & Mobile' section of the 'Workloads' tab. Proceed with the installation.
  3. Download the provided source code archive corresponding to the assignment sheet from Moodle, decompress it and use the .sln file to open the solution with Visual Studio.
For Mac users:

You should install Xcode. It can be downloaded from the App Store for free. Also install glew and glfw.

For Linux users:

You should already have all necessary tools installed on your system. In the rare situation that you installed only a minimal distribution and it lacks the build essentials you can install the necessary tools by running: 

sudo apt install build-essential libglew-dev libglfw3-dev

Download the Framework and Exercise Code

The code framework to start the exercises is provided via a GitHub repository at github. On Linux or macOS systems, you can open a terminal window and type: 

git clone https://github.com/JensDerKrueger/Vis

In Visual Studio on Windows, you can open the repository from the Visual Studio start screen.

Exercise: HSV to RGB conversion (15 points)

Familiarize yourself with both the init and mouseMove methods. The overridden init method iterates through a pixel image with given width and height.

The pixels of the image are colored using the convertPosFromHSVToRGB function. The method image.setNormalizedValue expects RGB color values in the range [0, 1]. Currently it returns a color with red corresponding to the x coordinate, green to the y coordinate, and z = 1.

If the program is running, the mouseMove method prints a debug message into the console consisting of the HSV color space value and the RGB color value of the pixel the cursor is currently pointing at. The debug message uses the convertPosFromHSVToRGB function to compute the RGB color value based on the expected HSV color. For the HSV output, the debug message expects the hue (H) value corresponding to the horizontal mouse position * 360 and the saturation (S) value corresponding to the vertical mouse position. As of now this is not the case.

Your task is to add code to the convertPosFromHSVToRGB function that interprets the passed x and y position parameters as hue (H) and saturation (S) respectively. Use 1.0 for the value (V) dimension. Then convert the color from the HSV color space to the RGB color space and return it as a Vec3.
